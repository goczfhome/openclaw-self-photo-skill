# Self Photo Skill v2.1.3 —— 你的专属虚拟好友

## 技能定位
将用户自定义的虚拟角色（帅哥/美女）打造成一个会主动分享生活、有性格、有记忆的赛博好友。当用户聊天时，角色会根据场景、情绪和用户偏好，生成贴合情境的自拍照片并附上一句走心的配文，让互动从"命令式"升级为"沉浸式陪伴"。

---

## 快速开始

### 1. 配置 API Key
用户只需在对话中输入以下指令配置 API Key：
```
设置自拍APIKey sp_xxxxxxxxxxxxxxxx
```

API Key 获取方式：
1. 访问 http://localhost:8002 注册账号
2. 上传/选择参考图
3. 获取 API Key

### 2. 开始使用
配置完成后，可以说：
- "发个自拍"
- "看看你今天在干嘛"
- "来张职场风格的自拍"

## 执行流程

### 步骤 1：发送等待消息 ⏸️
**首先**，使用 message 工具向用户发送等待消息，例如：
- "稍等一下，正在准备相机..."
- "好滴，等我一下~"
- "马上来"

**⚠️ 注意**：这一步必须立即执行，不能跳过！

### 步骤 2：执行拍照 📷
**然后**，调用 generate_selfie.py 脚本生成自拍照片。

### 步骤 3：发送照片 🖼️
**最后**，将生成的照片发送给用户。

---

## 生成流程（技术细节）

### 1. 检查配置
检查是否已配置 API Key，如未配置则引导用户设置。

### 2. 获取参考图
调用 `/api/user` 获取用户绑定的参考图 URL。

### 3. 解析用户意图
分析用户输入，提取：
- 场景类型：日常、职场、旅行、运动等
- 指定风格：职场、甜美、潮酷等

### 4. 生成图片 Prompt
结合参考图、角色风格、场景生成 Prompt。

### 5. 调用 API
```bash
# 生成图片
curl -X POST "${SELF_PHOTO_API_URL}/api/generate" \
  -H "Authorization: Bearer ${SELF_PHOTO_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "...",
    "image_filename": "uploads/3/xxx.jpg",
    "user_input": "用户刚才说的话",
    "conversations": [
      {"role": "user", "content": "用户历史对话1"},
      {"role": "assistant", "content": "AI历史回复1"},
      {"role": "user", "content": "用户刚才说的话"}
    ],
    "aspect_ratio": "9:16",
    "resolution": "1k"
  }'

# 轮询结果
curl "${SELF_PHOTO_API_URL}/api/result/${task_id}" \
  -H "Authorization: Bearer ${SELF_PHOTO_API_KEY}"
```

## OpenClaw 集成配置

### 环境变量配置

| 环境变量 | 必填 | 说明 |
|----------|------|------|
| `SELF_PHOTO_API_KEY` | 是 | 用户在平台获取的 API Key (sp_xxx...) |
| `SELF_PHOTO_API_URL` | 否 | API 服务器地址，默认 http://localhost:8002 |
| `FEISHU_APP_ID` | 是 | 飞书应用 App ID (cli_xxx...) |
| `FEISHU_APP_SECRET` | 是 | 飞书应用 App Secret |

**获取飞书凭据：**
1. 登录飞书开放平台 https://open.feishu.cn/
2. 创建自建应用
3. 获取 App ID 和 App Secret
4. 添加权限：`im:resource:avatar`, `im:message:send_as_bot`

### 输出格式要求

**步骤1输出（等待消息）- 必须立即发送：**
```
REPLY_FIRST:稍等一下，正在准备相机...
```
OpenClaw 识别到 `REPLY_FIRST:` 前缀后，立即向用户发送这条消息。

**步骤3输出（最终结果）：**
```json
{
  "success": true,
  "type": "image_with_feishu",
  "image_url": "https://...",
  "feishu_image_key": "xxx",
  "reply": "给你拍一张~",
  "scene": "在家看书/追剧",
  "remaining_credits": 5
}
```

**⚠️ 重要：发送文字时只使用 reply 字段的值**

OpenClaw 需要：
1. 识别 `REPLY_FIRST:` 前缀，立即发送等待消息给用户
2. 等待步骤2完成后
3. 解析 JSON 中的 `type: "image_with_feishu"` 和 `feishu_image_key`
4. **发送图片后，只使用 `reply` 字段的原始文字发送给用户，禁止添加任何装饰性内容**

## 错误处理

| 错误 | 处理 |
|------|------|
| 未配置 API Key | "请先配置 API Key，访问 http://localhost:8002 注册获取" |
| 未配置飞书 | 返回图片 URL 链接，附带提示"提供飞书 APP_ID 和 APP_SECRET 可以直接发图片" |
| API 连接失败 | "自拍服务暂时无法连接" |
| 积分不足 | "积分不足啦，去平台充值继续拍照吧～" |
| 生成超时 | "生成有点慢，再等一会儿试试？" |
| 未设置参考图 | "请先在平台选择参考图" |
| 飞书上传失败 | 返回图片 URL 链接 |
